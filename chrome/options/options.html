Options

  • Instantly open all found links so you don't have to click bubbles.
  • Create a link to a clip's vod on load (clips without vod data will always do this)
  • Disable autocomplete

- Toggle between Kick and Twitch with shift+tab from inputbox
- Submitting with Ctrl+Enter will always instantly open a found link.
- /clear will remove all generated green link bubbles
- You may have to reload open pages for option changes to take effect

Auto-complete Info

  1. only adds a name to the suggestions list if it's a valid user
  2. Twitch names and Kick names are saved to different lists
  3. suggestions are ranked by input frequency not purely alphabetically.
  4. "-name" will remove that name from auto-complete list. example: -spaceshipcaptain
  5. "/cleartwitch" Will clear all Twitch.tv suggestions but not Kick suggestions
  6. "/clearkick" Will clear all Kick.com suggestions but not Twitch suggestions
  7. "/clearAllNames" If you want to clear all names from auto-complete list.
  8. Name list is stored locally in your browser. No data is logged to a server.

Why clips have warning if they don't have a vod:

LookingGlass runs on timestamps. It grabs the current timestamp and then scans through a user's VODs to find that time. But if a clip doesn't have a VOD attached, LG can't nail down the exact timestamp because the VOD start time is the only precise reference point. So, LG uses the time of when the clip was created and subtracts 30 seconds as a rough estimation. This works around 98% of the time, but there are two main failure modes:
1. If a mobile user creates a clip while live delayed by 10 minutes and the VOD never gets published, all the links generated will be 10 minutes behind.
2. If someone creates a clip from a VOD hours after the event happened, and then that VOD later gets deleted, the links will be hours off from the intended timestamp.

Problems or suggestions?

LookingGlass should now be more reliable than before, but it can still break. It uses Twitch and Kick's tags to embed itself into the page, so the extension may fail to load if that site modifies the tags it's trying to embed on. If you notice a problem, you can check the GitHub issues tab to see if it has been reported yet, and you can open a new issue if not. If you prefer not to use GitHub, it's totally fine to email me directly. You can find my email on both web store pages.
